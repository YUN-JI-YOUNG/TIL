## HTML    

###### Hyper Text Markup Language/ 웹 컨텐츠의 의미, 구조를 정의할 때 사용하는 마크업 언어          

- `!` 이후 tab하면 기본정보는 자동완성     

### Hyper Text    

- hyper는 정보가 동일 선상에 있는 것이 아닌, 다중으로 연결된 상태 의미     

- 기존 웹의 문서는 다음페이지로의 이동만 가능했었지만, 획기적인 Hyper text의 개발로 참조를 통해 다른 문서로 즉시 접근할 수 있게 되었다.      
- Hyper text가 쓰인 기술들 중 가장 중요한 2가지는 http, html     

### Markup     

- 주제별로 단락을 나누고 글씨 크기를 그에 따라 조정하는 등 특정 텍스트에 역할(대주제, 본문, 소주제 등)을 부여하여 구조를 나누는 것

### Markup Language

- 문서나 데이터의 구조를 명시하는 언어     

- 다른 프로그래밍 언어처럼 저장, 조건, 반복 등을 사용할 수 없고, 단순히 데이터를 구조화하여 표현하기만 하는 언어     

- 웹 페이지를 작성하기 위한 것이며, 웹 컨텐츠의 의미와 구조를 정의

  ex> html, markdown

## CSS     

###### cascading style sheets (떨어지는 스타일 시트)  / 사용자에게 문서(HTML)를 표시하는 방법을 지정하는 언어          

- html 문서에 스타일과 레이아웃 적용       
- 주로 활용하는 속성 위주로 기억 -> 빈도가 낮은 속성은 필요할 때마다 검색하여 사용          
- 추가 내용은 [Learn CSS!](https://web.dev/learn/css/selectors/) 참고

#### 1. CSS 구문     

   - 선택자(스타일 적용할 요소)와 함께 시작 {속성:값 }     
   - 속성과 값의 한 쌍을 `선언문`이라고 함 -> 다양한 선언 가능      

 </br>     

#### 2. 정의 방법    

   - 인라인     
     - 직접 태그 안에서 스타일 속성 활용     
   - 내부 참조    
     - head 태그 내의 `<style>` 태그 내에 활용
   - 외부 참조      
     - link 태그를 이용하여 외부 CSS 파일을 head 태그 내에 불러오기     
     - 모듈화와 비슷 -> 반복 방지 가능     

 </br>

   (1) `!important` -> 선언문 뒤에 작성하면 해당 선언문은 무슨 짓을 해도 바뀌지 않음      

   ​	-> 특수한 경우 제외하곤 보통 사용하진 않음     

   (2) 인라인 > id 선택자 > class 선택자 > 요소 선택자      

   (3) 코드 순서

#### 4. CSS selector       

- 개발자 도구를 사용하여 선택자 추출 가능      

- 기본 선택자
  - 전체 선택자     
    - `*`{} 애스터리스크 = 전체 의미     
  - 요소 선택자     
    - `해당 html 요소 {}  `    
  - 클래스 선택자     
    - `.클래스{}` 해당 클래스가 적용된 모든 요소를 선택 -> 문서 곳곳에서 사용 가능하므로 반복 가능         
  - 아이디 선택자
    - `#id {}` 여러번 사용해도 동작은 하지만, 한 문서에선 1번만 사용(암묵적 약속)      
  - 속성 선택자  

 </br>     

- 결합자
  - 자손 결합자(`.box p`)
    - 공백으로 구분하며, 하위의 모든 요소 포함       
  - 자식 결합자 (`.box > p`)
    - 본인의 바로 하나 아래(들여쓰기 하나 차이)에 있는 특정 요소만 선택         
  - 일반 형제 결합자 (`p ~ span`)    
    - 형제 요소(같은 라인)중 본인보다 뒤에 있는 모든 요소 선택       
  - 인접 형제 결합자 (`p + span`)     
    - 형제 요소(같은 라인)중 본인 바로 뒤에 있는 특정 요소만 선택      

 </br>

#### 6. CSS 단위      

- 픽셀(px)      
  - 모니터 해상도의 한 화소인 픽셀 기준이며 고정적 단위 -> 각 모니터마다는 다를 수 있음        
- 퍼센트(%) 
  - 백분율 단위, 가변적인 레이아웃에서 주로 사용       
- em     
  - 바로 위, 부모 요소에 대한 상속의 영향을 받음 -> 부모가 변경되면 같이 변경          
    - 10px 에 2em이면 20px를 가짐           
  - 배수 단위, 요소에 지정된 사이즈에 상대적인 사이즈를 가짐        
- rem      
  - r = root        
  - 바로 위, 부모 요소에 대한 상속의 영향을 받지 않음      
  - 최상위 요소(html)의 사이즈를 기준으로 배수 단위를 가짐     
  - 고정적인 부모를 가지므로 em보다 사용에 용이       

```python
<style>
.em {
  font-size : 1.5em;
}
.rem {
  font-size : 1.5rem;
}
</style>

<body>
  <ul class = "em">
    <li class="em">1.5em</li>
    <li class="rem">1.5rem</li>
    <li>no class</li>
  </ul>

* html = 16px
결과 :
   1.5em  -> 36px(16*1.5*1.5) : 본인 em + 부모 em 
   1.5rem -> 24px(16*1.5) : 본인 rem만 
   no class -> 24px (16*1.5) : em,rem 없지만, 부모인 ul에 상속받아서  
```

#### 8. CSS Box Model     

- HTML의 모든 요소는 박스형태로 구성     

- 기본적으로 박스의 크기는 content-box로 설정되어있는데 원하는 사이즈로 만들기 위해선, border-box로 설정 변경해야 편리     

  `* { box-sizing: border-box;}` 를 스타일 맨 처음에 선언      

- 하나의 박스는 네 영역으로 이루어짐      

  - content    

    - 요소의 실제 내용     

  - padding    

    - 테두리와 실제 내용과의 여백    

  - border    

    - 테두리 영역       

    - width, style, color 등의 속성     

    - shorthand를 이용하여 순서 상관없이 한 번에 작성 가능   

      `border : 5px dashed blue` : "선 너비 5px, 점선, 파란색"라는 의미         

  - margin     

    - 테두리 바깥의 외부 여백       
    - `margin: 0 auto; ` : 방향이 상하/좌우이므로, 상하 여백이 없고 좌우 여백이 균등하므로 **가운데 정렬**     

- shorthand      

  - `margin:  5px ;`  상하 좌우(top, bottom, left, right)      
  - `margin:  5px 10px ;` 상하/좌우       
  - `margin:  5px 10px 20px ;` 상/좌우/하      
  - `margin:  5px 10px 20 px 30px ;` 상/우/하/좌       

- 마진 상쇄 (주의!)        

  - A의 top과 B의 bottom에 적용된 각각의 마진이 둘 중 큰 마진 값으로 결합되는 현상       
  - 그러므로 둘 중 하나의 값만 설정   
  - 좌우 마진은 상쇄 X

#### 9. CSS Display       

- 모든 요소들을 시각적으로 어떻게 보여줄지 결정하는 속성      

- 더 다양한 속성은 MDN에서 확인!         

- 인라인 / 블록 요소     
  - display: block       
    - 너비의 100%를 차지    
    - 줄 바꿈이 일어나는 요소    
    - 블록 레벨 요소 안에 인라인 레벨 요소가 들어갈 수 O      
    - div / ul, ol, li / p / hr / form 등     
    - 왼쪽 정렬 : margin-right : auto;
    - 오른쪽 정렬 : margin-left : auto;
    - 가운데 정렬 : 위의 둘다    
  - display: inline    
    - 너비를 본인 컨텐츠 내용만큼만 차지      
    - 줄 바꿈이 일어나지 않는 행의 일부 요소     
    - width, height, margin-top/bottom 지정할 수 없음     
    - 상하 여백은 line-height로 지정 (글자의 위아래 높이)      
    - span / a / img / imput, label / b, em, i, strong 등          
    - 왼쪽 정렬 : text-align : left;    
    - 오른쪽 정렬 : text-align : right;    
    - 가운데 정렬 : text-align : center;     
  - display: inline-block           
    - 블록과 인라인 레벨 요소의 특징을 모두 보유    
    - 인라인처럼 한 줄에 표시 가능 + 블록처럼 모든 속성 지정 가능     
  - display: none (주의!)    
    - 해당 요소를 화면에 표시하지 않고, 공간도 사라짐 -> 다른 레이아웃에 영향           
    - visibility hidden과 자주 비교되는데, hidden은 보이지 않을 뿐, 공간은 차지함         

 </br>     

#### 10. CSS position     

- 문서 상에서 요소를 배치하는 방법 지정     

- 모든 태그의 기본 위치 -> static     

  - 좌측 상단부터 배치    
  - 부모 요소 내에서 배치될 때는 부모 요소의 위치를 기준으로 좌측 상단부터 배치      

- 좌표 프로퍼티(top, bottom, left, right)를 사용하여 이동 가능 - 음수 가능       

  - relative - 상대적 위치(외출)    

    - 자신의 static 위치를 기준으로 이동     

    - 레이아웃에서 요소가 차지하는 공간은 static일 때와 동일 

      -> 과거 위치는 그대로 차지하는 중      

      -> 다른 box와 겹칠 수 있음          

    ```python
    .relative { 
      position : relative;    
      top : 100px; 
      left: 100px; }
    -> 기존 위치에서 위에서 100떨어지고 왼쪽에서 100떨어진 위치     
    ```

  - absolute - 절대적 위치(출가후 새로운 부모 찾기)     

    - 요소를 문서 흐름에서 제거 후 레이아웃에서 공간 차지 X       

      -> 다른 요소의 위치와 간섭하지 않는 격리된 사용자 인터페이스 기능을 만드는 데 활용   

      ex. 팝업 정보 상자, 제어 메뉴, 롤오버 패널, 페이지 어느 곳에서나 드래그할 수 있는 UI  등

    - static이 아닌 가장 가깝고, relative 혹은 absoulte인 부모/조상 요소를 기준으로 이동 

      -> 없으면 body에 붙는 형태         

      -> body에 붙지 않도록 absolute 사용하기 전, 부모를 만드는 것이 최우선!     

    ```python
    .parent {
      position : relative
    }
    
    .absolute-child { 
      position : absolute;    
      top : 100px; 
      left: 100px; }
      
    -> 부모 위치의 좌측 상단 꼭짓점을 기준으로 이동  
    ```

  - fixed - 고정 위치(출가)    

    - 요소를 문서 흐름에서 제거 후 레이아웃에서 공간 차지 X     
    - 부모요소와 관계없이 viewport 기준으로 이동     
    - 스크롤을 이동해도 항상 같은 곳에 위치    

</br>
